# Plan 01-01: Build Card, Button, Badge Primitives

**Phase**: 1 — UI Primitives
**Requirements**: UI-01, UI-02, UI-03

## Goal
Create reusable Card, Button, and Badge components in `client/src/components/ui/` that every page will import.

## Tasks

### Task 1: Create Card component
**File**: `client/src/components/ui/Card.jsx`
**What**: A wrapper component with consistent shadow, border-radius, padding, and overflow-hidden.
**Props**: `className`, `children`, `hover` (boolean, enables rc-card hover effect), `padding` (default `p-5`)
**Implementation**:
```jsx
import { cn } from '../../utils/cn';
export default function Card({ children, className, hover = false, padding = 'p-5' }) {
  return (
    <div className={cn(
      'bg-white rounded-2xl border border-slate-100 shadow-sm overflow-hidden',
      padding,
      hover && 'rc-card',
      className
    )}>
      {children}
    </div>
  );
}
```

### Task 2: Create cn() utility
**File**: `client/src/utils/cn.js`
**What**: Tiny utility combining clsx + tailwind-merge for className merging.
```js
import { clsx } from 'clsx';
import { twMerge } from 'tailwind-merge';
export function cn(...inputs) { return twMerge(clsx(inputs)); }
```

### Task 3: Create Button component
**File**: `client/src/components/ui/Button.jsx`
**What**: Button with `variant` (primary/secondary/danger/ghost), `size` (sm/md/lg), `loading`, `disabled`, `icon` props.
**Variants**:
- primary: `bg-teal-600 text-white hover:bg-teal-700`
- secondary: `bg-slate-100 text-slate-700 hover:bg-slate-200`
- danger: `bg-red-50 text-red-600 hover:bg-red-100`
- ghost: `text-slate-500 hover:bg-slate-50`
**Loading state**: Shows Loader2 spinner from lucide-react, disables button.

### Task 4: Create Badge component
**File**: `client/src/components/ui/Badge.jsx`
**What**: Small pill/tag for post types, match strength, statuses.
**Props**: `variant` (teal/blue/purple/amber/red/slate/green), `size` (sm/md), `children`, `className`, `dot` (boolean, shows colored dot before text)
**Color map**: Each variant maps to specific bg + text color classes.

## Acceptance Criteria
- [ ] `cn()` utility exists and works with conditional classes
- [ ] Card renders with consistent shadow, rounded-2xl, overflow-hidden
- [ ] Button renders 4 variants with visible style differences
- [ ] Button shows spinner when loading=true
- [ ] Badge renders colored pills for different variant values
- [ ] All components accept className for override
